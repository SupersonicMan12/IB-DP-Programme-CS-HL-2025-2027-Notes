### A1.3.2 Describe the functions of an operating system

Key term: System Integrity = state of a computer system when the system performs its intended functions correctly and reliably, without authorized or unintended alterations.
> [!EXAM TIP] IB-style question: what are the benefits of having a system of Integrity

A **Process** = an instance of a computer program that is being executed. 
Stages and states of a process:
![[Screenshot 2026-03-25 at 18.25.29.png]]

A Multi-user system has many users using the same machine / network, then processes need to be shared on the machine:
- Single Programming: many programs batched in one group, queued and loaded into memory one by one
- Multi Programming: two or more programs loaded into main memory, maximize the use of CPU time: use I/O time of A to run B
- Notice difference from Multitasking*
![[Screenshot 2026-03-25 at 19.01.55.png]]
Physical Address = actual address in the main memory; Virtual Memory = a virtual address that is logical that points to the physical memory. A table is created to map between the two.
- Virtual Memory is always always there to the system, but physical memory can change
	- Swapping = moving pages from RAM to disk storage temporarily (physical)
	- Excessive page swapping causes **thrashing** that damages system performance
**Paging**:
- In Physical Memory, Paging = the splitting of memory into small, equal sized sections
- In RAM / Main Memory, Frame = holds pages
**Segmentation**: notice this is conceptually parallel to paging
- Segments of different sizes

**File System** management: OS manages secondary storage through folder/directory structure
- Fragmentation occurs through creation, modification and deletion; Defragmentation is to rearrange back into contiguous

**Device Management**: 
**Device Drivers** are the OS's connection to each hardware component
**I/O** Methods:
- Buffering: temporary storage before I/O
- Caching: stores frequently accessed data to reduce access times
- Spooling: queues data for "queue devices" like printers
It is important for OS to verify authorization of devices to ensure integrity.

**Security**: provide firewalls, encryption, access control, user auth, etc.
**Accounting**: the tracking and recording of resource usage to make OS more informed
**GUI, Graphical User Interface**: user-friendliness
**Virtualization**: allowing multiple virtual machines to run on a single physical machine. ![[Screenshot 2026-03-25 at 19.44.25.png]]
**Networking**: provides services for data transmission and communication, while ensuring data integrity.

### A1.3.3 Compare different approaches to scheduling 

Recall processes, to maximize efficiency of CPU:
- First-come first-served: QUEUE
- Round Robin: each process is assigned a fixed time slice / quantum, and the machine cycles in 123...n 123...n etc.
- Priority Scheduling: hierarchy of priority + first-come-first served
- Multilevel queue scheduling: each priority has its own queue, and the queues may use different scheduling methods

### A1.3.4 Evaluate the Use of Polling and Interrupt Handling

**Interrupt** = a signal to the processor indicating an event needs immediate attention.
- Suspend current activity, save state, executing Interrupt Service Routine (ISR)
- Examples: paper unavailable, peripheral services, power problem
- Passive Notification
**Polling** = a technique where the processor checks status of a device at regular intervals to check for attention
- Active checking

### A1.3.5  Explain the role of the operating system in managing multitasking and resource allocation

**Multitasking** is similar to multiprogramming, except that it's "processes" are "tasks" that are smaller than processes. This means the small individual processes share smaller resources, e.g. one CPU.

Challenges of multitasking and resource allocation:
- Task scheduling: 
	- **fairness**, hierarchy while ensuring **starvation** doesn't happen
	- **pre-emption**, suspending process for another higher-priority one
- Resource contention:
	- Deadlock = each process needs a resource held by another, hence system gets stuck. 
		- OS must release the deadlock,
		- or perform **prevention** proactively
	- Livelock = constantly requesting and releasing resources, never making progress
	- Thrashing = excessive page swapping due to insufficient memory (?? needs more expl.)

### A1.3.6 Describe the use of the control system components

Control system — any system that manages, commands, or regulates the behavior of another system or process. e.g. Vending Machine. 
- Input, process, output
- Feedback mechanism:
	- Open-loop: output had no influence on input
	- Closed-loop: output's feedback affects the way input is processed in the future, self-correcting and more complex and expensive.

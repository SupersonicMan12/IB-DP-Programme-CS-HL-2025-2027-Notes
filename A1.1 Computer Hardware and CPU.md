## CPU Structure and Machine Cycle (1, 5, 6)

##### Parts of CPU (脑子)

Von Neumann Architecture: input, cpu, output.

**Units**: Arithmetic Logic Unit (ALU) and Control Unit (CU)
**Registers**:
- Instruction Register (IR) - current instruction
- Program Counter (PC) - address of next instruction
- Memory Address Register (MAR) - address of memory being accessed
- Memory Data Register (MDR) - data being read from or written to memory
- Accumulator (AC) - intermediate arithmetic/logic results
**Buses**:
- Address Bus - carries memory addresses
- Data Bus - carries data
- Control Bus - carries instructions 
**Processor**:
- Single-core processor - one instruction at a time
- Multi-core processor - parallel execution
- Co-processor - specialized secondary processor (graphics)

##### Fetch–Decode–Execute Cycle 

Timed and synchronized by **clock**: a frequency for operations (in GHz)

**Fetch**:
- PC holds next address
- MAR receives the address from PC
- CU controls: 
	- address bus carries address from MAR
	- control bus carries signal to read
	- data bus returns data or instruction
- MDR holds the fetches data or instruction
- IR stores the instruction for decoding
**Decode**:
- CU reads instruction from IR
- Decodes via CPU instruction --> Opcode + Operand
**Execute**:
- ALU performs operation, returning to AC or memory

##### Pipelining

Instead of this:

| F1  | D1  | E1  |     |     |     |
| --- | --- | --- | --- | --- | --- |
|     |     |     | F2  | D2  | E2  |
|     |     |     |     |     |     |

Do this:

| F1  | D1  | E1  |     |     |
| --- | --- | --- | --- | --- |
|     | F2  | D2  | E2  |     |
|     |     | F3  | D3  | E3  |

This is within a **single core**. Across multiple cores, we call that **parallelism**.

Advantage: faster processing

## Types of Primary Memory (4)

CPU + Registers are not memory

From closest to farthest to CPU: L1 < L2 < L3 < RAM = ROM < Secondary (roughly)

L1 - L3 are Caches, or the "frequently accessed memory"
L1 to L3: faster in speed, smaller in size
RAM = random access memory
ROM = read only memory, for initialization etc.

SRAM = faster but more expensive than DRAM; Caches are SRAM, main RAM is DRAM

When a CPU needs something, first checks L1, then L2, then L3, then RAM

**Cache miss** = when the fetch's target is not in cache, meaning it looks in RAM, leading to longer fetch time and slower cycles

**Cache hit** = opposite, good


> [!NOTE] From S1M2:
> E.g. Effect of Cache Size? 
> Greater size smaller probability of **Cache Miss**, greater probability of **Cache Hit**, hence shorter fetch time and faster cycles

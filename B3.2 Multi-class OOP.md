```java
class Tank{
	protected int hp, at, mv;
	private String password;
	public Tank(int hp, int at, int mv){
		this.hp = hp;
		this.at = at;
		this.mv = mv;
		this.password = "secret";
	}
	public void attack(Tank B){
		B.getHit(at);
	}
	public void getHit(int dmg){
		hp -= dmg;
	}
	public String getType(){
		return "Tank";
	}
}

class HeavyTank extends Tank{
	public HeavyTank(int level){
		if (level == 1){
			super(380, 45, 5);
		} else if (level == 2) {
			super(520, 55, 6);
		}
	}
	public void getHit(int dmg){
		dmg /= 2;
		super.getHit(dmg);
	}
	public String getType(){
		return "Heavy Tank";
	}
	public String getType(int x){
		String s;
		for (int i = 1; i <= x; i++){
			s += "Heavy Tank ";
		}
		return s;
	}
}
```

Identified:
- `HeavyTank` **inherits** `Tank`, `HeavyTank` is a subclass of the superclass `Tank`
- Subclass calls `super()` and has its very own constructor
- **Polymorphism**: 
	- Static Polymorphism (Overloading): `getType()` and `getType(int x)`, in same class
	- Dynamic Polymorphism (Overriding): getType() between the two classes, super and sub, dynamic binding or late binding
		- Notice that `final` cannot be overriden
		- Overriding variables = shadowing variables, should be avoided
- Advantages of Polymorphism:
	- Increases reusability and flexibility of code
	- Maximizes code maintainability and straightforwardness
	- Easy extensibility
- `Protected` state to ensure only access by subclass, `Private crush` is not open to `HeavyTank` for some unknown reason

Ways of Dynamic Polymorphism:
- Inheritance
- Interfaces

Widening assignment is allowed:
```java
Superclass a = new Subclass();
a.method(b);
```
At compile-time, b from Superclass is found - static binding
- hence if no b in Superclass, error!
At run-time, b goes to Subclass - dynamic binding

Narrowing assignment is not allowed

Class relationships:
- Inheritance: A is a B 
- Dependency: A uses a B or A depends on B, invoking methods of the other
- Aggregation: A has a B, contains references to other objects as instance data
- Composition: A has a B, and A owns lifecycle of B! Stronger Aggregation
ID go A to B, AC have the diamond at A, the owner!

```java
interface Armored{
	// abstract method
	void getHit(int dmg);
}
// notice by default it is public and abstract (unimplemented)

abstract class AntiMine{
	abstract public void getHit(int dmg);
}

class HeavyTank extends Tank implements Armored{
	public void getHit(int dmg){
		...
	}
}
```

- An abstract method is a method header without implementation. 
- Interfaces have default public + abstract.
- Interface cannot be instantiated
- Class implements an interface by:
	- stating `implements ...`
	- providing implementation for every abstract class
- An abstract class!
	- not all its methods are abstract, does not default abstract or public

Notice a class can implement multiple interfaces but can only inherit one class.

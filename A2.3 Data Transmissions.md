### A2.3.1 Describe different types of IP addressing
- The distinction between IPv4 and IPv6 addressing
- The differences between public IP addresses and private IP addresses, and between static IP addresses and dynamic IP addresses 
- The role of network address translation (NAT) to minimize the use of IP addresses and to facilitate communication between private internal networks and the public internet

IPv4: 
![[Screenshot 2026-04-16 at 08.59.03.png]]
Prefixes: 0, 10, 110 (length l), then network prefix has length 8l, the rest is host.


Public IP Address:
- Managed by IANA, needs registration; no collisions allowed
- Connected to internet directly

Private IP Address:
- Computers not connected to the internet, need not have globally unique IP addresses, only through TCP/IP
- Use private IP address, network address translation (NAT) to translate to a public IP address that everyone in the LAN shares
- Private addresses look like:
	- A (8 prefix), B (12 prefix), C (16 prefix)
- Allows for duplicates globally

NAT - network address translation:
- Cause: not enough IPv4, IPv6 gives alternative, but is slow and costly to be adopted
- Solution: use a single or a few public addresses to encompass a bunch of private IP addresses

Ultimate purpose = **Abstraction** = the operating system hides low-level hardware complexity and gives the user/programmer simpler logical interfaces to work with. This means:
- Communicating with peripherals
- Coordinating concurrent processing of jobs
- Memory, program and data management
- Resource monitoring and accounting
- Security
- Network managing
- Providing an appropriate UI to the User

### A1.3.2 Describe the functions of an operating system
- Maintaining system integrity while running operating systems’ background operations 
- Memory management, file system, device management, scheduling, security, accounting, graphical user interface (GUI), virtualization, networking

#### Processor Management
A **Process** = an instance of a computer program that is being executed. 
Stages and states of a process:
![[Screenshot 2026-03-25 at 18.25.29.png]]

- Single Programming = one program is in main memory and running at a time; if it waits for I/O, the CPU sits idle
- Multi Programming: two or more programs loaded into main memory, maximize the use of CPU time: use I/O time of A to run B
- Notice difference from Multitasking*

#### Memory Management
![[Screenshot 2026-03-25 at 19.01.55.png]]
Physical Address = actual address in the main memory; Virtual Memory = a virtual address space mapped to physical memory (an abstraction of the physical memory). A table is created to map between the two.
- Swapping = moving pages from RAM to disk storage temporarily (physical)
- Thrashing = excessive swapping that damages system performance
**Paging**:
- In Physical Memory, Paging = the splitting of memory into **small, equal sized sections**
- In RAM / Main Memory, Frame = holds pages
**Segmentation**: notice this is conceptually parallel to paging
- Segments of **different sizes** according to logical division

##### File System
OS manages secondary storage through folder/directory structure
- Fragmentation is the splitting and scattering of the same file across different parts of the disk
- Defragmentation is to rearrange back into contiguous
##### Device Management: 
**Device Drivers** are the OS's connection to each hardware component
**I/O** Methods:
- Buffering: temporary storage before I/O
- Caching: stores frequently accessed data to reduce access times
	- How does this relate to IO: consider I/O paths that the CPU accesses
- Spooling: queues data for "queue devices" like printers
It is important for OS to verify authorization of devices to ensure integrity.

Other Functions:

**Security**: provide firewalls, encryption, access control, user auth, etc.
**Accounting**: the tracking and recording of resource usage to make OS more informed
**GUI, Graphical User Interface**: user-friendliness
**Virtualization**: allowing multiple virtual machines to run on a single physical machine. ![[Screenshot 2026-03-25 at 19.44.25.png]]
> [!NOTE] Memo Tip: Virtualization != Virtual Memory
> Virtualization abstracts the compter
> Virtual Memory abstracts the memory

Advantages:
- The sharing of multiple VMs of a single machine allows for efficient use of computing resources
- Allows for flexibility of OS given the same computer
- Allows for remote working
- Isolation ensures security of each virtual machine

**Networking**: provides services for data transmission and communication, while ensuring data integrity.

### A1.3.3 Compare different approaches to scheduling 

Recall processes, to maximize efficiency of CPU:
- First-come first-served: QUEUE
- Round Robin: each process is assigned a fixed time slice / quantum, and the machine cycles in 123...n 123...n etc.
- Priority Scheduling: hierarchy of priority + first-come-first served
- Multilevel queue scheduling: each priority has its own queue, and the queues may use different scheduling methods

> [!NOTE] Difference: Time Sharing vs. Slicing
> Time sharing is the idea of multiple tasks sharing the CPU
> Time slicing is the mechanism of time-interval division through which time sharing is achieved
### A1.3.4 Evaluate the Use of Polling and Interrupt Handling
- Event frequency, CPU processing overheads, power source (battery or mains), event predictability, controlled latency, security concerns 
- Real-world scenarios may include keyboard and mouse inputs, network communications, disk input/ output operations, embedded systems, real-time systems.

**Interrupt** = a signal to the processor indicating an event needs immediate attention.
- Suspend current activity, save state, executing Interrupt Service Routine (ISR)
- Examples: paper unavailable, peripheral services, power problem
- Passive Notification
**Polling** = a technique where the processor checks status of a device at regular intervals to check for attention
- Active checking

Considerations to determine which is better:
- Event frequency: if it is very frequent, polling is better
- CPU overhead: polling wastes CPU cycles repeatedly checking
- Power source: interrupts are usually better-powered devices so the CPU can sleep
- Event predictability: similar to event frequency, if it's predictable and scheduled checks needed
- Controlled latency, Security Concerns

### A1.3.5  Explain the role of the operating system in managing multitasking and resource allocation
- The challenges of multitasking and resource allocation, including task scheduling, resource contention and deadlock

**Multitasking** = multiple tasks/processes sharing CPU to give simultaneous user experience

> [!NOTE] Difference from Multiprogramming
> Multiprogramming is about keeping the CPU busy and maximizing its utilization
> Multitasking is about simultaneity and responsiveness

Challenges of multitasking and resource allocation:
- Task scheduling: 
	- **Fairness**, hierarchy while ensuring **starvation** doesn't happen
	- **Pre-emption**, suspending process for another higher-priority one
		- **Starvation** = waits an unreasonably long time due to priority on other processes
- Resource contention, multiple processes compete for the same limited resource
	- Deadlock = each process needs a resource held by another, hence system gets stuck. 
		- OS must release the deadlock,
		- or perform **prevention** proactively
	- Livelock = constantly requesting and releasing resources, never making progress
	- Thrashing, as previously defined

> - Memory Management: the OS allocates memory to each task, ensuring processes do not interfere with each other and enabling multiple tasks to run concurrently.
> - Processor Management: through time slicing/sharing, interrupt and polling, ensures a smooth user experience

### A1.3.6 Describe the use of the control system components

Control system — any system that **manages**, commands, or regulates the behavior of another system or process. e.g. Vending Machine. 
- Input, process, output
- Feedback mechanism:
	- Open-loop: output had no influence on input
	- Closed-loop: output's feedback affects the way input is processed in the future, self-correcting and more complex and expensive.

Nice vocab:
- Sensors / Transducers: both convert analog signals to digital data usable by the control system.

### A2.2.2 Describe the function of servers. (HL only) 
- Types of servers: domain name server (DNS), dynamic host configuration protocol (DHCP), file server, mail server, proxy server, web server 
- Factors to consider must include function, scalability, reliability and security. 

Domain Name Server (DNS): translates human-readable URL / domain names into corresponding numerical IP addresses

Dynamic Host Configuration Protocol (DHCP): automatic assignment of IP addresses, check static vs dynamic IP addresses (discussed in A2.1)

File Server: centralized location to store, access and manage files; FTP and SFTP are protocols used here

Mail Server: for managing emails; SMTP, POP3, IMAP

Proxy Server: intermediary between user and internet, literally a "proxy":
- masking user's real IP address
- block access to specific websites
- store copies of frequently accessed

Web Server: literally what runs the website

Systematic advantages: 
- Scalability: workload is distributed across large demand
- Reliability: systematic, yet if server fails then fails
- Security: secured against cyber-attacks

### A2.2.3 Compare and contrast networking models. 
- Client-server and peer-to-peer models 
- The respective benefits and drawbacks of client-server and peer-to-peer models
- Real-world applications may include web browsing, email services, online banking, file sharing, VoIP services, blockchain.

**Network Models**:
- Client-server model: client requests network service, server provides it (central command)
	- Benefits: centralized control, scalability, efficiency, security
	- Drawbacks: single point of failure, risk; cost at the center
	- Web browsing, email, online banking
- Peer-to-peer model
	- Benefits: decentralized risk, cost effective, scalability, direct sharing
	- Drawbacks: coordination (e.g. synchronization), reliability
	- BitTorrent file sharing, Voice over IP services, Blockchain

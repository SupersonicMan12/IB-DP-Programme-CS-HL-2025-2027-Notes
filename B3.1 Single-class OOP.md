An object has 
- Attributes: who am I?
- Methods: what can I do?

Class is the blueprint of object, objects are instances of classes
```java
class Hans{
	private int age, size;
	public Hans(int age, int size){
		this.age = age;
		this.size = size;
	}
	public void bark(int x){
		for (int i = 1; i <= x; i++)
		System.out.println("WOOF!\n");
	}
	public int get_age(){
		return age;
	}
	public void set_age(int age){
		this.age = age;
	}
}

public class Main {
    public static void main(String[] args) {
	    Hans hans = new Hans(3, 69);
		hans.bark(100);
    }
}
```

notice the **Instantiation** of

```java
Hans hans = void Hans(3, 69);
```

Where `hans` is the object and `Hans` is the class. Upon instantiation, the object is **initialized** according to the class's **constructor**.

This also follows **encapsulation**, where variables are hidden under `private` and methods are `public`.

What if there's something true or useful to all Hans's?

```java
public Hans{
	private int age, size;
	static private boolean awesome;
	public Hans(int age, int size){
		this.age = age;
		this.size = size;
		awesome = true;
	}
	public void bark(int x){
		for (int i = 1; i <= x; i++)
		System.out.println("WOOF!\n");
	}
	public int get_age(){
		return age;
	}
	public void set_age(int age){
		this.age = age;
	}
	public boolean is_awesome(){
		return awesome;
	}
}

public class Main {
    public static void main(String[] args) {
	    Hans hans = new Hans(3, 69);
		hans.bark(100);
		System.out.println(hans.is_awesome());
    }
}
```

That's a static variable `awesome` and static method `is_awesome`:
- Shared across all objects of a single class
- Uses space only per the class, not once for each instantiated object

Perfect example: calculators have static methods, as math functions should be all-same; some classes feel "static", like calculators.
